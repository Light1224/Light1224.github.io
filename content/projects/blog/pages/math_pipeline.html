Math Pipeline

This page is the exhaustive technical record of the full project pipeline, including repository coverage, phase coverage, formulas, diagnostics, artifacts, and reproducibility closure.

Repository-wide coverage map

The pipeline is implemented in a repository that includes modeling code, preprocessors, notebooks, scripts, tests, reports, and a large artifact tree. The top-level components reflected here are:

  • Code packages: estimators, preprocessing, spectral, rmt, model, regimes, validation.
  • Execution scripts: run_load_qa.py, run_preprocess.py, run_spectrum_rmt.py, run_regimes.py, run_validation.py, run_all_pipeline.py plus audit/report scripts.
  • Artifacts root folders: transformed, spectra, rmt, model, regimes, validation, plots, regime_ab.
  • Reporting layer: report/all_phases_overview.txt, phase metric files, exhaustive compilation report, and integrated findings report.

The all_phases_overview summary tracks 467 generated files and phase-specific output volumes. This matters because conclusions are tied to pipeline outputs, not a single notebook snapshot.

Notation and core objects

Assets are indexed by i = 1..N, time by t = 1..T. The project often peaks at N=76 in full-cross-section settings with varying effective T by strictness and windowing. Core symbols used throughout this page are:

  • \(\sigma_{i,t}\): realized volatility proxy
  • \(x_{i,t}\): standardized log-vol panel entry
  • \(C\): correlation matrix
  • \(C = V \Lambda V^\top\): eigendecomposition
  • \(K\): retained mode count
  • \(F\): factor score matrix
  • \(B\): loading/eigenvector matrix

Full phase chain (from baseline through phase 27)

Baseline to phase 8: preprocessing base

Pre-phase output establishes that raw realized volatility is right-skewed with occasional spikes, requiring transform/stabilization before spectral work.

Phase 9: correlation geometry and QA

Matrix diagnostics establish a valid decomposition target in strict full-sample form: symmetry error near zero, PSD checks passing in selected constructions, trace consistency, and broad positive co-movement structure. Group structure and connectivity profiles are extracted from pairwise behavior and cluster-like cohorts.

Phase 10: eigenspectrum concentration and low-rank behavior

Core spectral results: strong top-mode concentration with nontrivial residual dimensionality; low-rank projection stabilizes geometry but does not fully explain full matrix behavior. Rolling windows show improved stability with longer windows.

Phase 11 and 12: random-matrix separation

Marchenko-Pastur support is used to classify outlier modes versus bulk. Under baseline settings, a small set of significant modes sits above MP edge while most eigenvalues remain in bulk-like range. Variant sensitivity confirms concentration shifts and lower-tail pathologies in some smoothed constructions.

q = N/T
lambda_min = (1 - sqrt(q))^2
lambda_max = (1 + sqrt(q))^2

Phase 13 and 14: factor panel and interpretation

Selected modes are projected into factor panel. Interpretation metrics include market-vol correlation, liquidity coupling, ACF persistence, and sector concentration (HHI). The first factor behaves as systemic anchor; second factor is more concentrated and conditional.

Phase 15 and 16: stability and filtered reconstruction

Rolling overlap and principal angles show persistent stability hierarchy: PC1 high stability, PC2 lower stability with rotation episodes. Filtered reconstruction preserves structural core and improves numerical usability of matrix operations.

Phase 17: low-rank model residual anatomy

Low-rank modeling quantifies systematic versus idiosyncratic share and verifies residual cross-correlation reduction. This stage clarifies what common factors can and cannot explain at asset level.

Phase 18, 19, 20, 21: regime feature engineering, segmentation, and validation

Composite signal and feature construction feed change-point segmentation through penalty scan and BIC-guided selection. Segment usefulness is validated with regime-level volatility, dispersion, factor-share, and drawdown behavior, followed by split robustness checks.

Phase 22 and 23: estimator sensitivity and falsification

Estimator swap (RS vs GK) maintains core dimensionality and regime-level structure with mild concentration shifts. Randomization tests (time and stock shuffles) are used to challenge structural claims under null settings.

Phase 24, 25, 26: pipeline hardening and final reproducibility

Script-level reproducibility checks confirm required runners exist and deterministic audit passes in documented outputs. Final reports are assembled from generated tables/figures rather than manually typed metrics.

Phase 27: strict-causality predictive diagnostics

This stage enforces timing-integrity constraints:

  • strict forward target definition (future-only windows)
  • walk-forward PCA signal generation (no full-sample reuse at prediction time)
  • train-only event thresholding before test evaluation
Target_h(t) = mean( market_vol_{t+1}, ..., market_vol_{t+h} )
Predictors_t = { factor_energy_t, rolling_angle_mean_t }

From report metrics: aligned rows ≈ 5,081, best test AUC ≈ 0.9007 (factor energy, horizon 10), best test R² remains negative in linear OOS fit (≈ -0.313 at best).

Complete formula block used across stages

Rogers-Satchell:
RS_t = sqrt( ln(H_t/C_t) ln(H_t/O_t) + ln(L_t/C_t) ln(L_t/O_t) )

Garman-Klass:
GK_t = sqrt( 0.5[ln(H_t/L_t)]^2 - (2ln2 - 1)[ln(C_t/O_t)]^2 )

Log transform and z-score:
y_{i,t} = ln(sigma_{i,t})
x_{i,t} = (y_{i,t} - mu_i)/s_i

Correlation:
C_{ij} = corr_t(x_i, x_j)

Eigendecomposition:
C = V Lambda V^T

Top-K reconstruction:
C^(K) = sum_{k=1..K} lambda_k v_k v_k^T

Reconstruction error:
E_K = ||C - C^(K)||_F / ||C||_F

MP bounds:
q = N/T
lambda_min = (1 - sqrt(q))^2
lambda_max = (1 + sqrt(q))^2

Factor projection:
F = X B_K

Rolling geometry:
angle_t = arccos(|v_{1,t}^T v_{1,t-1}|)
overlap_t = |v_{1,t}^T v_{1,t-1}|

Low-rank model:
X_t = B f_t + epsilon_t

Factor energy:
energy_t = sum_k f_{k,t}^2
