Scope and Data

This post defines the project boundaries before any interpretation is attempted.

The central question is straightforward: is cross-sectional volatility structure genuinely compressible and stable, or does it only appear meaningful because of sample noise? To answer that, the project is constrained by fixed panel construction rules, staged diagnostics, and full metric exports that allow each phase to fail independently.

The workload is not a small notebook run. Artifact inventory reports show hundreds of generated files, with phase tables ranging from moderate row counts to very large aggregate outputs. That scale matters because every narrative claim can be traced back to reproducible artifacts.

What is treated as ground truth

The target is not directional return prediction. The target is the geometry and evolution of realized-volatility structure across assets. Core objects are:

  • standardized log-volatility panel
  • cross-sectional correlation matrix geometry
  • eigenspectrum shape and MP separation
  • factor dynamics, overlap drift, and angle rotation
  • regime segmentation quality and validation behavior
  • strict-causality predictive diagnostics for forward risk events

Primary report sources used in this write-up

  • report/all_phases_overview.txt for global output scale
  • report/phase_15_metrics.txt for stability stress and outlier-drop tests
  • report/phase_19_metrics.txt for regime count/penalty diagnostics
  • report/phase_27_metrics.txt for strict-causality predictive outcomes
  • findings_report.md for full storyline and earlier interpretations

Data reality check

The panel spans long history, but effective coverage varies by window and strictness setting. In strict phase-27 outputs, the aligned dataset has 5,081 rows while active asset counts still fluctuate by window. For that reason, conclusions are framed as persistent low-dimensional tendency with regime sensitivity, not as a single invariant dimensionality claim.
