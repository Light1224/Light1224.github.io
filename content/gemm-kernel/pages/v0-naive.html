GEMM v0: Naive Triple Loop Baseline

Version 0 (gemm/v0_naive.cpp) is the reference kernel used to lock down correctness and establish the lower performance bound for the rest of the stack. Its implementation is intentionally direct: one output element at a time, one dot product at a time, no data transformation, no architecture-specific intrinsics, and no threading.

Kernel structure and data access

The loop nest is i-j-k. For each output coordinate (i, j), the kernel accumulates sum += A[i*lda + k] * B[k*ldb + j] across k and writes exactly one scalar to C[i*ldc + j]. This means there is no reuse of partial sums across neighboring columns or rows and no attempt to preserve a tile in registers. The algorithmic complexity is still O(MNK), but the memory behavior is the key limiter.

What this version is good for

  • Ground truth behavior for future refactors.
  • Simple place to verify stride usage (lda, ldb, ldc).
  • Baseline for speedup calculations against blocked, packed, and parallel kernels.
  • Debug-friendly control flow with minimal hidden state.

Known limitations in this code path

  • No explicit cache blocking, so working-set growth quickly hurts locality.
  • No SIMD microkernel, so scalar issue width dominates throughput.
  • No panel packing, so B accesses repeatedly traverse original layout.
  • No thread-level parallelism, so large matrices are single-core bound.

Benchmark harness alignment

tests/benchmark_gemm_v0.cpp uses the same measurement helper as other versions (time_best in tests/benchmark_common.hpp). The current default run set is intentionally reduced for v0 to keep runtime bounded, with an exhaustive mode available via ATLAS_BENCH_V0_FULL=1. That keeps v0 practical in full-suite test runs while preserving opt-in deep sweeps.

How v0 anchors the optimization ladder

Every later version changes one major axis: loop order (v1), cache blocking (v2), scalar micro-tiling (v3), SIMD kernels (v4), packed panels (v5), parallel scheduling (v6), and policy dispatch (v7). v0 remains the simplest specification of "what GEMM computes" and therefore the safest fallback reference during tuning.
