Falsification and Reproducibility

Entry: this is where I tried to break my own signal and verify the package can be rerun cleanly.

I always think this phase is where a project becomes either credible or decorative. If signals survive only in one handcrafted run with soft assumptions, I can’t trust them. So I used randomization tests, estimator swaps, and reproducibility audits as a deliberate “try to kill it” sequence.

Randomization falsification

Time-shuffle is the strongest kill switch in my outputs: structure collapses hard under temporal randomization. Stock-shuffle behaves differently and is less absolute, which I consider informative rather than embarrassing because it tells me there is strong global coupling that remains hard to erase under some null constructions.

Estimator robustness

The RS vs GK rerun confirms the big-picture conclusions are stable: same effective signal count in baseline comparisons and small concentration shifts. I like this because it lowers model-risk dependence on one estimator definition.

Reproducibility hardening

The deterministic double-run audit passed with fixed seed and matching inventories/hashes in documented checks. This matters to me because it means this project can be re-executed without hidden manual edits.

My blunt summary

I did not get a “perfect pass everywhere” fairy tale. I got a realistic empirical profile: strong temporal structure, robust systemic mode, harder edge cases for some nulls, and fully reproducible packaging. I trust that profile more than I would trust perfection.
